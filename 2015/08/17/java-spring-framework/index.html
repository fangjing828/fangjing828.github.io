为什么要学习框架？框架带来了什么？

Web 程序发展变化史

  1. Servlet
  2. JSP
  3. 框架

    Servlet

servlet 在 web.xml 中配置。

JSP

框架

  • 配置
  • 注解
    Struct 解决了 web 开发中的职责分配问题，实现了显示与逻辑分离。

    Web 开发需要什么

  1. 数据
  2. 页面 (客户端生产与消费数据) 信息
  3. 业务 (服务端生产与消费数据）信息
    NOTE:信息用于消除不确定性

数据 (M, Model) 在页面 （V, View) 与 业务（C, Controller) 中流动带来的问题：

  1. 数据的序列化与反序列化问题。
  2. 序列化效率(json, xml, protobuffer)
  3. 数据格式与内容的校验。
  4. 数据如何在页面展示。
  5. 数据请求从 View 层到 Controller 层。如何知道请求调用时那个类，那个方法？ Http Request 又如何在 Control 层的 Java 建立关系。

Controll 层还需要关心的问题

  1. 作为逻辑处理程序的 facade 门面，如果逻辑处理程序发生异常，该如何处理？
  2. 对于逻辑处理的结果，我需要做什么样的处理，才能满足丰富的前台展示需要。

这一个又一个问题的提出，都基于对MVC的基本概念的挖掘。所以，这些问题都需要我们在写程序的时候去一一解决。说到这里，这篇文章开头所提的问题应该可以有答案了：框架是为了解决一个又一个在Web开发中所遇到的问题而诞生的。不同的框架，都是为了解决不同的问题，但是对于程序员而言，他们只是jar包而已。框架的优缺点的评论，也完全取决于其对问题解决程度和解决方式的优雅性的评论。所以，千万不要为了学习框架而学习框架，而是要为了解决问题而学习框架，这才是一个程序员的正确学习之道。
