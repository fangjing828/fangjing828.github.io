java-spring-listener

综述

当 web 应用运行于 web 容器时，会产生各种事件（如 web 启动、停止、用户 session 创建、销毁、请求到达等。

Listener

Listener 接口 Event 类
ServletContextListener ServletContextEvent
ServletContextAttributeListener ServletContextAttributeEvent
HttpSessionListener HttpSessionEvent
HttpSessionActivationListener HttpSessionEvent
HttpSessionAttributeListener HttpSessionBindingEvent
HttpSessionBindingListener HttpSessionBindingEvent
HttpRequestListener ServletRequestEvent
ServletRequestAttributeListener ServletRequestAttributeEvent

ServletContextListener

  • 接口方法：contextInitialized(ServletContextEvent e);contextDestroyed(ServletContextEvent e);
  • 触发： 在 web 容器加载 web 应用程序时（如启动），触发 contextInitialized 方法。在 web 容器移除 web 应用程序时，触发 contextDestroyed 方法。

    ServletContextAttributeListener

  • 接口方法：attributeAdded(ServletContextAttributeEvent e);attributeRemoved(ServletContextAttributeEvent e);attributeReplaced(ServletContextAttributeEvent e);
  • 触发：若有对象加入为 application(ServletContext) 对象属性，会触发 attributeAdded，反之若有属性变化则触发 attributeReplaced 与 attributeRemoved

    HttpSessionListener

  • 接口方法：sessionCreated、sessionDestroyed。
  • 触发: session(HttpSession) 对象创建或过期时，会分别触发 sessionCreated 与 sessionDestroyed 方法。

    HttpSessionAttributeListener

  • 接口方法：attributeAdded;attributeReplaced;attributeRemoved.
  • 触发 session(HttpSession) 对象属性变化时，则会触发上述对应的方法。

    HttpSessionBindingListener

  • 接口方法：valueBound;valueUnBound.
  • 触发: value 从 session(HttpSession) 中加入或者删除时触发。

    HttpSessionActivationListener

  • 接口方法：sessionDidActivate;sessionWillPassivate.
  • 触发： Activate与Passivate是用于置换对象的动作，当session对象为了资源利用或负载平衡等原因而必须暂时储存至硬盘或其它储存器时(透过对象序列化)，所作的动作称之为Passivate，而硬盘或储存器上的session对象重新加载JVM时所采的动作称之为Activate，所以容易理解的，sessionDidActivate()与 sessionWillPassivate()分别于Activeate后与将Passivate前呼叫。

    ServletRequestListener

  • 接口方法：requestInitialized;requestDestroyed.
  • 触发：request(HttpServletRequest)创建与释放时触发事件。

    ServletRequestAttributeListener

  • 接口方法:attributeAdded;attributeReplaced;attributeRemoved.
  • 触发：有对象加入为request(HttpServletRequest)对象属性变更时会触发上述方法。
