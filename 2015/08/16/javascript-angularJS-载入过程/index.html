javascript-angularJS-载入过程

preparation

浏览器载入 HTML，解析成 DOM.

Process

  1. 浏览器载入 angular.js.
  2. DOMContentLoaded 事件触发 AngularJS 执行。
  3. AngularJS 需找 ng-app 指令所指示的应用边界(module)。
  4. ng-app
  5. 使用 ng-app 指定模块配置注入器 ($injector)
  6. $injector 用于创建编译服务($compile service) 和 根作用域($rootScope)
  7. ng-init 指令将 ”word“ 赋值给作用域中变量 name。
  8. 表达式替换。

module

module 是 angular 代码入口，需要先声明 module, 才可以定义其他组件元素。angular 的 module 封装了 controller、service、filter、directive、config、run 等组件。

NOTE:module 声明创建有且只有一次，获取已声明的 module 可以有多次。
angular 模块加载器 setupModuleLoader(window)
angular 在启动时，会设置全局的 angular 对象，然后在 angular 对象上发布 module 这个API。关于 module API 代码，能清晰的看见第一行谓语句，module 名称不能以 hasOwnProperty 命名，否则会抛出 ”badname“ 的错误信息。紧接着，如果传入了name 参数，其表示是声明 module，则会删除已有的module信息，将其置为null。

从moduleInstance的定义，我们能够看出，angular.module为我们公开的API有：invokeQueue、runBlocks、requires、name、provider、factory、servic、value、constant、animation、filter、controller、directive、config、run。其中invokeQueue 和 runBlocks 是按名约定的私有属性，请不要随意使用，其他 API 都是我们常用的 angular 组件定义方法，从invokeLater 代码中能看到这类 angular 组件定义的返回依然是 moduleInstance 实例，这就形成了流畅 API，推荐使用链式定义这些组件，而不是声明一个全局的 module 变量。

最后，如果传入了第三个参数 configFn，则会将它配置到 config 信息中，当 angular 进入 config 阶段时，它们将会依次执行，进行对 angular 应用或者 angular 组件如 service 等的实例化前的配置。
