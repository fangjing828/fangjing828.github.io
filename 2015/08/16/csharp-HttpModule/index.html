inetinfo.exe: www 服务进程,IIS 服务与 ASPNET_ISAPI.dll 都寄存在此进程中。
ASPNET_ISAPI.dll：是处理 .apsx 文件的 win32 组件。IIS 服务器只能识别 .html 文件，当服务器发现请求的文件是 .aspx 文件时，IIS 服务器将文件转交给 ASPNET_ISAPI.dll 处理。
*ASPNET_WP.exe:ASP.NET 框架进程提供 .NET 运行的托管环境(NET CLR, common language running time)。

ASP.NET 请求处理过程是基于管道模型。管道模型由多个 HttpMoudle 与 HttpHandler 组成。HttpRequest 请求被传递给各个 HttpMoudle,最终由某个 HttpHandler 处理。处理完成后，再沿原路径返回 HttpRequest 请求结果。

NOTE:HttpRequest 请求过程中，可以调用一个 HttpHandler，但可以调用多个 HttpMoudle。

Init(HttpApplication context) ： 允许 HttpMoudle 向 HttpApplication 对象中注册自定义事件处理程序。
