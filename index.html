java-spring-listener

综述

当 web 应用运行于 web 容器时，会产生各种事件（如 web 启动、停止、用户 session 创建、销毁、请求到达等。

Listener

Listener 接口 Event 类
ServletContextListener ServletContextEvent
ServletContextAttributeListener ServletContextAttributeEvent
HttpSessionListener HttpSessionEvent
HttpSessionActivationListener HttpSessionEvent
HttpSessionAttributeListener HttpSessionBindingEvent
HttpSessionBindingListener HttpSessionBindingEvent
HttpRequestListener ServletRequestEvent
ServletRequestAttributeListener ServletRequestAttributeEvent

ServletContextListener

  • 接口方法：contextInitialized(ServletContextEvent e);contextDestroyed(ServletContextEvent e);
  • 触发： 在 web 容器加载 web 应用程序时（如启动），触发 contextInitialized 方法。在 web 容器移除 web 应用程序时，触发 contextDestroyed 方法。

    ServletContextAttributeListener

  • 接口方法：attributeAdded(ServletContextAttributeEvent e);attributeRemoved(ServletContextAttributeEvent e);attributeReplaced(ServletContextAttributeEvent e);
  • 触发：若有对象加入为 application(ServletContext) 对象属性，会触发 attributeAdded，反之若有属性变化则触发 attributeReplaced 与 attributeRemoved

    HttpSessionListener

  • 接口方法：sessionCreated、sessionDestroyed。
  • 触发: session(HttpSession) 对象创建或过期时，会分别触发 sessionCreated 与 sessionDestroyed 方法。

    HttpSessionAttributeListener

  • 接口方法：attributeAdded;attributeReplaced;attributeRemoved.
  • 触发 session(HttpSession) 对象属性变化时，则会触发上述对应的方法。

    HttpSessionBindingListener

  • 接口方法：valueBound;valueUnBound.
  • 触发: value 从 session(HttpSession) 中加入或者删除时触发。

    HttpSessionActivationListener

  • 接口方法：sessionDidActivate;sessionWillPassivate.
  • 触发： Activate与Passivate是用于置换对象的动作，当session对象为了资源利用或负载平衡等原因而必须暂时储存至硬盘或其它储存器时(透过对象序列化)，所作的动作称之为Passivate，而硬盘或储存器上的session对象重新加载JVM时所采的动作称之为Activate，所以容易理解的，sessionDidActivate()与 sessionWillPassivate()分别于Activeate后与将Passivate前呼叫。

    ServletRequestListener

  • 接口方法：requestInitialized;requestDestroyed.
  • 触发：request(HttpServletRequest)创建与释放时触发事件。

    ServletRequestAttributeListener

  • 接口方法:attributeAdded;attributeReplaced;attributeRemoved.
  • 触发：有对象加入为request(HttpServletRequest)对象属性变更时会触发上述方法。

java-spring-jsp

综述

JSP(Java Server Page, Java 服务器页面），其本质上是一个简化的 Servlet 设计。JSP 技术与 ASP 技术类似，是在 HTML 插入 Java 程序段(Scriptlet) 与 JSP 标记（tag),从而形成 JSP 文件。

JSP 工作过程

  1. client 发起 HTTP Request。
  2. server 对 HTTP Request 进行处理。
  3. 如果该 jsp 页面第一次被请求，会编译成 servlet。
  4. servelt 动态生成 html 页面返回 client。
    NOTE: Servlet 是单例，所有的 client 访问的是同一个 servlet。

    JSP 文件组成

  5. 静态数据，如 HTML 文件。
  6. JSP 指令，如 include 指令。
  7. JSP 脚本元素和变量。
  8. JSP 动作。
  9. 用户自定义标签。

    静态数据

    输入文件中的内容和输出给 HTTP 响应的内容完全一致，一般是指纯 HTML 页面。

    JSP 指令

    控制 JSP 编译器如何生成 servlet,例如：
指令 意义
import java 导入声明
contentType 规定生成内容的类型。当生成 HTML 内容或当前字符集 character set 并非默认字符集时使用
errorPage 处理 HTTP 请求时，如果出现异常则显示该错误提示信息页面
isErrorPage 如果设置为 TRUE,则表示当前文件时一个错误提示页面
isThreadSafe 表示最终生成的 servlet 是否是线程安全的

JSP 脚本元素与变量

标准脚本变量：

  1. out - JSPWriter 用来写入响应流的数据
  2. page - servlet 自身
  3. pageContext - 一个 PageContext 实例包括和整个页面相联系的数据。一个给定 HTML 页面在多个 JSP 之间传递。
  4. request - HTTP request 对象
  5. response - HTTP response 对象
  6. session - session 对象
    脚本元素： 插入 Java 代码到 jsp servlet.

  7. 声明标签 - 变量定义，可定义静态数据成员。

  8. 脚本标签 - java servlet 类 _jspService() 方法中放入包含的语句。

  9. 表达式标签

JSP 动作

JSP 动作是一系列可以调用内建于网络服务器功能中的 XML 标签。

JSP 内置对象

request.getRequestDispatcher（’a.jsp’）.forward(request,response)

response 对象包含了响应客户请求的有关信息

java-thread-pool

综述

线程池用于控制当前线程执行数量。线程池作用：

  1. 减少创建和销毁线程的次数。
  2. 控制当前线程并发数目。
class function
ExecutorService 真正的线程池接口
ScheduledExecutorService 类似 Timer / TimerTask，用于解决重复执行的问题
ThreadPoolExecutor ExecutorService 默认实现
ScheduledThreadPoolExecutor 继承 ThreadPoolExecutor 的 ScheduleExecutorService 接口实现，周期性任务调度类的实现

线程池实现

  1. ThreadPoolExecutor
parameter name note
corePoolSize 线程池中保存的线程数，包括空闲线程
maximumPoolSize 线程池中允许的最大线程数
keepAliveTime 当线程数大于核心时，多余的空闲线程等待新任务的最长时间
unit keeepAliveTime 参数时间单位
workQueue 执行前用于保持任务的队列， 此队列仅保存由 execute 方法提交的 Runnable 任务
threadFactory 执行程序创建线程时使用的工厂
handler 超出线程范围和队列容量而执行被阻塞时所使用的处理程序
ThreadPoolExecutor Executor 类的底层实现

  1. newSingleThreadPool 实现： 固定大小线程池

  2. newFixedThreadPool 实现： 单线程

  3. newCachedThreadPool 实现： 无界线程池，可以进行自动线程回收
    无界线程池， maximumPoolSize

java 修饰符 Sychronized

在多线程程序中，应当尽可能使用线程安全的集合。java 线程安全的集合主要包括： ConcurrentLinkedQueue、ConcurrentHashMap、ConcurrentSkipListMap、ConcurrentSkipListSet 等(java.util.concurrent)。常用非同步集合如下:

  1. ArrayList 与 LinkedList
  2. HashSet 与 LinkedHandSet
  3. HashMap
  4. LinkedBlockingQueue
    创建同步的非同步集合

常用的同步集合：Vector 与 HashTable。
